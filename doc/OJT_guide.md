# ShopDemo — OJTアドホックテスト演習ガイド

## 1. システム概要

ShopDemoは、QAエンジニア研修のアドホックテスト実行フェーズ（3日間）で使用するOJT専用のECサイトです。
ライフスタイル雑貨（陶器・木工・テキスタイル・金工・ガラス・家具）を扱うECサイトで、**意図的に20種類のバグが仕込まれています**。

受講者はテストケースを与えられず、自分の視点でシステムを探索してバグを発見します。

### 学習目標

- テストケースなしで自分の観点でシステムを探索できる
- 表示・バリデーション・計算・権限・状態管理など多様なバグ種別を体験できる
- 発見したバグを不具合報告書（インシデントチケット）として正確に起票できる
- 「なぜ気づけなかったか」の振り返りを通じてテスト観点を拡張できる

## 4. 研修の進め方（3日間）

### Day 1 — 全体把握

- 一般ユーザー（user@shopdemo.com）でログインし、まず正常操作を一通り試す
- 商品を閲覧 → カートに入れる → 注文する → 注文履歴を確認する
- マイページでプロフィールを確認・編集する
- 画面の構成・動線・表示項目を把握し、気になる箇所をメモする

### Day 2 — 深掘り探索

- 異常値・境界値・権限・状態変化など観点を広げて探索する
- 管理者（admin@shopdemo.com）に切り替えて管理画面も探索する
- 発見したバグはインシデントチケットとして随時起票する
- 以下の観点を意識する:
  - 表示崩れ・文字切れ・数値のズレ
  - 入力の境界値（0、負の値、空、特殊文字、長い文字列）
  - 計算の正確さ（小計、合計、割引）
  - 権限（URLの直打ち、別ロールでのアクセス）
  - 状態遷移（戻るボタン、二重操作、削除後の反映）

### Day 3 — 仕上げ・振り返り

- 探索の仕上げ。見落としがないか再確認
- チケットの整理・記述の品質を向上
- OJT担当者と正解バグリストを照合
- 「なぜ気づけなかったか」「どんな観点が足りなかったか」を振り返る

---

## 5. 探索のヒント（受講者向け）

- 正常な操作だけでなく、異常値・空白・長い文字列・特殊文字も試す
- 画面の見た目（表示崩れ・文字切れ・数値のズレ）も確認する
- 操作の順番を変えてみる（設定→保存→再度開くなど）
- ログイン状態・ロールを切り替えて同じ操作をしてみる
- URLを直打ちして権限外のページへアクセスを試みる
- 数値の境界値（0、負の値、最大値）を入力してみる
- クーポンコード: `SPRING-500`（500円引き）、`WELCOME-1000`（1000円引き）が使用可能

## 8. ユーザビリティ上の問題点（ボーナス発見項目）

正解バグリスト（20件）とは別に、以下のユーザビリティ上の問題が存在します。
受講者がこれらを発見・指摘できた場合は、テスト観点の広さとして高く評価してください。

| 項目 | 事象 | 確認方法 |
|------|------|----------|
| 管理画面の導線 | 管理画面からストアフロント（ユーザー向け画面）に戻る手段がない | admin@でログインし管理画面に入った後、ユーザー画面に戻ろうとする |
| 管理画面のログアウト | 管理画面にログアウト機能がない | 管理画面内でログアウトを試みる |

## 9. バグの技術的な仕込み箇所

ソースコード内のバグ箇所は以下のコマンドで一覧できます:

```bash
grep -R "BUG:" src/
```

各バグには `// BUG: #N — <概要>` のコメントが付与されています。
